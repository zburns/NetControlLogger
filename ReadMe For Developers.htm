Source Code is in C# (Version 2.0 - ECMA Specificatiosn)

External Component Dependancies by Developer Express (mostly for Grid and Lookup controls)

Developed and committed using Visual Studio 2010 SP1

Tried to keep "worst-case scenario" while developing.  So no reliance on Internet connectivity, Microsoft Office Products, etc.  If you're in a disaster scenario and just have a netbook, this program will work.

CodePlex Login Information is username/password....snd/username_cp and password (codeplex can act a little funny sometimes)

Why did I choose CodePlex vs. SourceForge, GoogleCode, etc, etc.  I was originally looking for a cross-platform source control system such as Subversion or Git that I could house on GoDaddy or some other shared server.  I always hate having a third party kludge and the version control system (Team Foundation Server) works very well for Visual Studio integration.  Since it is a Windows product and probably always will be, I chose CodePlex because I needed it to seemlessly interact with Visual Studio, I could web-browse the source code, and use Subverson or Mercurial if need be.  No other public source control server seemed to handle Team Foundation Server connectivity.  Plus, I needed a stable company to back the source control store.  That only left SourceForge, Google, and CodePlex (Microsoft).
